# lywsd03mmc client

Python library and utilities for Xiaomi Mi Temperature and Humidity Monitor 2 / LYWSD03MMC sensors using the [bleak](https://pypi.org/project/bleak/) Bluetooth Low Energy library.

The library is based on [LYWSD02](https://github.com/h4/lywsd02) and [lywsd03mmc](https://github.com/uduncanu/lywsd03mmc).

In addition to reading current sensor values, this repository includes:

* `scripts/export_history.py` to export the hourly history stored on the sensor to CSV.
* `scripts/mi_dashboard.py` to explore the exported values in an interactive dashboard.

## Installation

Python `>=3.8, <3.13` is required.

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

For the dashboard, Python >=3.10 and the following additional dependencies are required. They can either be added to `requirements.txt` or installed separately:

```bash
pip install pandas plotly streamlit
```

## Get the device identifier

The scripts use a BLE device identifier. Its format depends on the operating system.

### Linux and Windows

Find the MAC address of the device in the Xiaomi Home app:

1. Open the device details.
2. Tap the three dots to open the settings.
3. Open **About** near the top of the list.
4. Copy the shown MAC address.

Example:

```python
SENSOR_IDENTIFIER = "A4:C1:38:12:34:56"
```

### macOS

On macOS, Core Bluetooth does not expose the sensor's MAC address. Run:

```bash
python scripts/discover_devices.py
```

Example output:

![discover.png](pics%2Fdiscover.png)

Use the discovered UUID as the device identifier:

```python
SENSOR_IDENTIFIER = "70C40C24-C60B-BB9D-D737-9895C5DA52F3"
```

The UUID is assigned by Core Bluetooth and is not an intrinsic property of the sensor. It may change; if a device can no longer be found, run device discovery again.

## Read current sensor values

### Sync example

```python
from lywsd03mmc.lywsd03mmc_client import Lywsd03mmcClientSyncContext, Lywsd03mmcData

SENSOR_IDENTIFIER = "70C40C24-C60B-BB9D-D737-9895C5DA52F3"

while True:
    try:
        with Lywsd03mmcClientSyncContext(SENSOR_IDENTIFIER, timeout_sec=60) as client:
            data: Lywsd03mmcData = client.get_data()
            print(data)
            print()
    except Exception as exception:
        print(f"Error: {exception}")
```

### Async example

```python
import asyncio

from lywsd03mmc.lywsd03mmc_client import Lywsd03mmcClient

SENSOR_IDENTIFIER = "70C40C24-C60B-BB9D-D737-9895C5DA52F3"


async def main(identifier: str) -> None:
    async with Lywsd03mmcClient(identifier, timeout=60) as client:
        data = await client.get_data()
        print(data)


asyncio.run(main(SENSOR_IDENTIFIER))
```

## Export stored history to CSV

The sensor stores hourly summary records. Each record contains:

* timestamp;
* minimum and maximum temperature during the hour;
* minimum and maximum relative humidity during the hour.

The export does **not** contain every instantaneous display reading.

### Configure the sensor identifier

Open `scripts/export_history.py` and set `SENSOR_IDENTIFIER`:

```python
# macOS: Core Bluetooth UUID
SENSOR_IDENTIFIER = "70C40C24-C60B-BB9D-D737-9895C5DA52F3"

# Linux or Windows: normally the Bluetooth MAC address
SENSOR_IDENTIFIER = "A4:C1:38:12:34:56"
```

### Run the export

Run the script from the repository root:

```bash
python scripts/export_history.py
```

The export is written to:

```text
mi_history.csv
```

The script:

* scans for the configured BLE device before connecting;
* reads history in small batches to make long BLE transfers more robust;
* can be started again after an interrupted export and keeps already exported records;
* reads the sensor's internal clock and corrects timestamps relative to the computer clock;
* retains the uncorrected device timestamp in `timestamp_device` for traceability;
* reports possible clock jumps in the stored data.

The script only reads the sensor timestamp. It does not change the device clock or delete stored history.

> Before exporting valuable local history, do not factory-reset the sensor. A reset may erase measurements that have not already been synchronised elsewhere.

### CSV columns

```text
idx
```

Record index supplied by the sensor.

```text
timestamp_device
```

Raw timestamp stored by the device.

```text
timestamp_corrected
```

Timestamp corrected using the offset between the device clock and the computer clock at export time.

```text
datetime_local
```

Human-readable corrected local date and time.

```text
temperature_min_c, temperature_max_c
```

Minimum and maximum temperature in degrees Celsius for the recorded hour.

```text
humidity_min_percent, humidity_max_percent
```

Minimum and maximum relative humidity in percent for the recorded hour.

## Explore exported history in the dashboard

The dashboard reads `mi_history.csv` from the repository root and does not communicate with the sensor.

Start it from the repository root:

```bash
streamlit run scripts/mi_dashboard.py
```

The dashboard provides:

* temperature and humidity timelines with hourly min/max bands;
* hourly, daily and weekly aggregation;
* minimum, average and maximum summary values;
* a temperature distribution chart;
* a temperature-versus-humidity scatter plot;
* detection of missing hourly records;
* a raw data table and filtered CSV download.

The dashboard expects timestamps corrected by `scripts/export_history.py`. If an older CSV still contains dates around 1970, rerun the export script so that the existing rows are migrated to corrected timestamps.

## Lywsd03mmc bluetooth services and characteristics
To list services and characteristics for a sensor, run:

```bash
python scripts/list_gatt.py
```

The relevant service is:

```text
ebe0ccb0-7a0a-4b0c-8a1a-6ff2997da3a6
```

### `EBE0CCC1-7A0A-4B0C-8A1A-6FF2997DA3A6` — Current temperature, humidity and battery voltage

Properties: `[Read, Notify]`

```python
print(client.get_data())
# temperature_row: 2479, temperature: 24.79, hum: 33,
# battery_vol: 2994, battery_percentage: 89
```

### `EBE0CCC4-7A0A-4B0C-8A1A-6FF2997DA3A6` — Battery value

Properties: `[Read]`

```python
print(client.get_battery())  # 3001
```

### `EBE0CCBE-7A0A-4B0C-8A1A-6FF2997DA3A6` — Temperature unit

Properties: `[Read, Write]`

```python
print(client.get_temp_unit())  # C
client.set_fahrenheit()
print(client.get_temp_unit())  # F
client.set_celsius()
print(client.get_temp_unit())  # C
```

### `EBE0CCB7-7A0A-4B0C-8A1A-6FF2997DA3A6` — Timestamp

Properties: `[Read, Write]`

```python
import datetime

print(datetime.datetime.fromtimestamp(client.get_timestamp()))
client.set_timestamp((datetime.datetime.now() - datetime.timedelta(days=1)).timestamp())
print(datetime.datetime.fromtimestamp(client.get_timestamp()))
```

`scripts/export_history.py` reads this characteristic to correct exported timestamps, but does not write to it.

### `EBE0CCB9-7A0A-4B0C-8A1A-6FF2997DA3A6` — Latest calculated hour record and status value

Properties: `[Read]`

```python
print(client.get_last_calculated_hour_idx_and_next_idx())  # (107, 108)
```

### `EBE0CCBB-7A0A-4B0C-8A1A-6FF2997DA3A6` — Last calculated hour data

Properties: `[Read]`

```python
print(client.get_last_hour_data())
# idx_num: 107, timestamp: 1702803600,
# temperature_row_max: 21.7, humidity_max: 39,
# temperature_row_min: 20.6, humidity_min: 37
```

### `EBE0CCBA-7A0A-4B0C-8A1A-6FF2997DA3A6` — First history record index

Properties: `[Read, Write]`

```python
print(client.get_first_history_idx())  # 0
client.set_first_history_idx(100)
print(client.get_first_history_idx())  # 100
```

### `EBE0CCBC-7A0A-4B0C-8A1A-6FF2997DA3A6` — History data stream

Properties: `[Notify]`

```python
client.set_first_history_idx(100)
print(client.get_first_history_idx())
print(client.get_last_calculated_hour_idx_and_next_idx())
print(len(client.get_history_data()))
```
